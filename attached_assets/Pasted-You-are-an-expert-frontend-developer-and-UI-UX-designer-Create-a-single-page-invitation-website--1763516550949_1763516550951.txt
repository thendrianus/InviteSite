You are an expert frontend developer and UI/UX designer.
Create a **single-page invitation website** (`index.html` only) for a **Christmas event**, designed in the elegant style of a **wedding invitation**.

The website must be:

* **Elegant**
* **Simple**
* **Center-oriented**
* **Mobile-first, extremely responsive**
* **Wedding invitation style**
* **Inspired heavily by the brochure design**
* With **starry night background like the brochure**
* Only one file: `index.html`
* Using HTML + inline CSS + minimal vanilla JavaScript
* No external frameworks (Bootstrap, Tailwind, etc.)

---

# 1. **Design & Style Requirements**

### Visual Style:

* Use a **starry night background** similar to the brochure:

  * Either using a provided image (if available), *or*
  * Using CSS stars effect (white/gold small glowing dots) blended with a **midnight blue** gradient background.
* Colors:

  * **Midnight blue / dark navy** as main color.
  * **Gold** accents for borders, headings, and buttons.
  * **White / off-white** for text.
* Typography:

  * Title “Tidak Ada yang Mustahil” → **cursive/script font** (wedding style).
  * Other text → **clean sans-serif**.
* Center-oriented layout:

  * All major sections centered.
  * Max-width container of **600–700px** centered on desktop.

### Mobile:

* Designed primarily for **mobile screens**.
* Perfect spacing, readable fonts, large touch targets.

### Starry Background Requirement:

* The hero section (top) must feature:

  * A **starry night effect**, either:

    * A high-quality starry sky background image, or
    * CSS-rendered stars (tiny lights randomly placed).
* Starry effect should be subtle, elegant, NOT distracting.
* Full-screen hero height with stars visible clearly.

# 4. **Dynamic Personalization (URL Parameters)**

Use URL parameters:

```
?to=Guest+Name&from=Sender+Name
```

### Behavior:

### Case A — `to` AND `from` exist:

```
Kepada Yth.
Bapak/Ibu/Sdr/i <Guest Name>

Undangan ini dikirim oleh:
<Sender Name>
```

### Case B — only `to` exists:

```
Kepada Yth.
Bapak/Ibu/Sdr/i <Guest Name>
```

(Hide sender block.)

### Case C — only `from` exists:

```
Kepada Yth.
Saudara/i terkasih

Undangan ini dikirim oleh:
<Sender Name>
```

### Case D — neither exists:

Default greeting:

```
Kepada Yth.
Saudara/i terkasih
```

(Hide sender block.)

PLUS:

* If `to` exists → autofill RSVP name field.
